¿Qué es CommonJobs?

  • Administración de empleados
  • Administración de postulantes
  • Búsquedas de trabajo públicas
  • Contabilización de vacaciones
  • Contabilización de ausencias

¿Por qué es útil?

  • Información centralizada
  • ¡No más papeles en la oficina!
  • Colaboración en el deparamento de RRHH
  • Seguimiento histórico sin recurrir a la memoria
  • Herramienta de discusión: provee una plataforma con la información para resolver problemas

Características especiales

Funcionalidad

Características especiales: funcionalidad

Legajo electrónico

Mantener legajo y documentación de todos los empleados es un proceso arduo. Mucho proceso manual es requerido y la consolidación con documentación física es desafiante.

CommonJobs presenta un legajo electrónico que se combina con perfiles de usuarios individuales. Esto permite el seguimiento personalizado y grupal de los empleados.

Características especiales: funcionalidad

Seguimiento de empleados y postulantes

Cada persona recibe un perfil para mantener toda su información agrupada y actualizada.

Estos perfiles constan de secciones independientes que permiten enfocarse en aspectos particulares de la relación empresarial con esta persona.

Cada perfil comienza con un resumen simple de la persona.

Búsquedas laborales

CommonJobs permite crear búsquedas de trabajo que al publicadas estarán disponibles en internet para la postulación.

Estas postulaciones automáticamente se convierten en postulantes a ser seguidos en el banco de postulantes.

Búsquedas laborales

Cada búsqueda laboral sugerirá los postulantes más adecuados para ocupar el puesto de acuerdo a las aptitudes que dichos postulantes posean.

Contabilización de ausencias

Las vacaciones son calculadas automáticamente.

Tecnología

Características especiales: tecnología

Seguridad desacoplada

Las clases que identifican a los usuarios y sus permisos están abstraídas de los sistemas en los que recaen. Esto significa que es simple agregar, quitar o cambiar el comportamiento del sistetma de seguridad sin tener que incurrir en grandes cambios en el sistema.

Dicho cambio ya se afrontó una vez durante el desarrollo, en donde se eliminó el uso de Active Directory para ser reemplazado por permisos de usuarios en bases de datos.

Para más detalles, ver la documentación de proyecto sobre el diseño de la autenticación y autorización.

public JsonNetResult AbsenceBunch(int year, BaseSearchParameters parameters)
{
    var query = new SearchAbsences(parameters)
    {
        From = new DateTime(year, 1, 1),
        To = new DateTime(year, 12, 31)
    };
    var results = Query(query);
    return Json(new
    {
        Items = results,
        Skipped = parameters.Skip,
        TotalResults = query.Stats.TotalResults
    });
}
